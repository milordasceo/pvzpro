# API — принципы

- Контракты описываются кратко и проверяются тестами.
- Для разработки используем моки данных и изолированные окружения.
- Все интеграции проходят через единые слои: `services/`, `store/`.

## Окружения
- dev: локальная разработка (моки)
- test: проверка контрактов (моки/стаб)
- prod: реальные сервисы (после верификации)

## Схемы данных
- Минимальные JSON‑схемы для основных сущностей (позже будет дополнено).

## Проверка контрактов
- Юнит‑тесты на парсинг и валидацию схем.
- Минимальные тестовые сценарии в `docs/test-specs/`.
